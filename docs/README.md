# Documentation Index

The documentation tree is grouped by purpose so the normative specs stay easy to
find while the toolkit guides live under `docs/apseudo-docs/`.

## Layout

| Folder | Purpose |
| --- | --- |
| [`docs/apseudo-docs/`](apseudo-docs/) | User-facing toolkit docs, including usage, features, enforcement, examples, and roadmap material. |
| [`docs/apseudo-docs/usage/`](apseudo-docs/usage/) | Command usage, install instructions, editor setup, testing instructions, agent wording, use-case guides, and operator guides. |
| [`docs/apseudo-docs/features/`](apseudo-docs/features/) | Feature-specific implementation and usage references for formatter, LSP, autocomplete, MCP, hooks, and skills. |
| [`docs/apseudo-docs/enforcement/`](apseudo-docs/enforcement/) | Enforcement-specific guides for linter, pre-commit, CI, and agent-runtime checks. |
| [`docs/apseudo-docs/examples/`](apseudo-docs/examples/) | Example `.apseudo` files, Markdown fenced blocks, and executable runner scripts. |
| [`docs/apseudo-docs/roadmap/`](apseudo-docs/roadmap/) | Future-version and deferred-work documents. |
| [`docs/reviews/`](reviews/) | Project review outputs, traceability reviews, and feature-gap analyses. |
| [`docs/reference/`](reference/) | Normative language and executable-runner specifications, rule catalog, language token/scope specs, source register, and background reference material. |
