# Scratchpad

We need to add:

- Define who owns specific docs or how that is determined
- Define who is the consumer of specific docs or how that is determined
- Define how to determine the status of a doc (draft, stable, deprecated, etc.); what triggers a state change?
- Define how to determine the confidence of a doc (unknown, low, medium, high); what triggers a state change?
- Provide a list of tags for each doc, and define what each tag means; does not need to necessarilly preclude the use of other tags, but should provide a clear definition of what each tag means and when it should be used.
- Provide rules for when to add and how to reference related docs, source docs, etc.
- Provide a set of common aliases for docs, and define when to use.
